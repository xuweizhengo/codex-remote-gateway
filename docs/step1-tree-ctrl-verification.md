# Step 1: DataViewTreeCtrl 可行性验证

## 目标

验证 wxdragon 的 `DataViewTreeCtrl` 是否满足 Agent 调用链追踪的需求。

**这是 Agent Manager MVP 的第一步，也是风险最高的一步。**

---

## 为什么这是第一步？

1. **风险最高** - TreeCtrl 在项目中未见实际使用案例
2. **影响设计** - 决定调用链 UI 的实现方式
3. **零依赖** - 可以独立测试，不需要数据库或现有代码
4. **耗时短** - 1-2 小时即可得出结论

---

## 测试点

| # | 测试项 | 预期结果 | 验证方法 |
|---|--------|---------|---------|
| 1 | 创建树形结构 | 显示多层节点 | 初始化时创建根节点+子节点 |
| 2 | 展开/折叠 | 点击可展开/折叠 | 手动点击节点前的箭头 |
| 3 | 点击事件 | 能选中节点 | 点击节点，观察是否高亮 |
| 4 | 实时添加节点 | 动态添加子节点 | 点击"添加子节点"按钮 |
| 5 | 更新节点状态 | 修改文本/图标 | 点击"更新状态"按钮 |
| 6 | 性能测试 | 100+ 节点流畅 | 批量添加节点 |

**适用场景**：

- ❌ 5+ 层深度嵌套

## 常见问题

### Q1: 如果编译错误 "DataViewTreeCtrl not found"？

**A**: 这说明 wxdragon 没有提供这个 API，立即切换到备选方案。

### Q2: 如果运行时崩溃？

**A**: 检查是否是 API 使用错误，尝试修复。如果无法修复，切换到备选方案。

### Q3: 如果功能不完整（如无法更新节点文本）？

**A**: 评估缺失功能的影响：
- 如果是次要功能（如图标） → 可以继续使用
- 如果是核心功能（如添加节点） → 切换到备选方案

### Q4: 备选方案能否满足需求？

**A**: 对于 Agent Manager MVP 的 2-3 层调用链，完全够用。Phase 2 可以考虑：
- 自定义绘制树形视图（wxDC）
- 嵌入 WebView + HTML/JavaScript 树形组件
- 等待 wxdragon 更新
